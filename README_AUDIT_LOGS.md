# เพิ่มเติม — หน้าบันทึกการใช้งานระบบ (Audit Logs Viewer)

ส่วนสุดท้ายของสเปคเดิมที่ยังไม่มี UI — ตอนนี้ทำครบแล้วครับ

## สิ่งที่เพิ่มเข้ามา

หน้า `/[locale]/audit-logs` (**Developer เท่านั้น** — บล็อก 2 ชั้นเหมือนหน้าจัดการผู้ใช้: redirect ที่ server component + RLS ที่ฐานข้อมูลที่ตั้งไว้ตั้งแต่ Phase 1):

- แสดงทุกการเปลี่ยนแปลงข้อมูลในระบบ: **เข้าสู่ระบบ, ออกจากระบบ, เข้าสู่ระบบไม่สำเร็จ, สร้าง/แก้ไข/ลบข้อมูล**
- กรองตาม **การกระทำ** และ **ตาราง** (vehicles, customers, contracts, payments, auth.users)
- ค้นหาตามรหัสข้อมูล (record ID)
- **คลิกแต่ละแถวเพื่อดู "ค่าเดิม" และ "ค่าใหม่"** แบบ JSON เทียบกันได้ทันที — ตอบคำถาม "ใครแก้ราคารถคันนี้ เมื่อไหร่ เปลี่ยนจากเท่าไหร่เป็นเท่าไหร่" ได้ในที่เดียว
- Pagination 30 รายการ/หน้า

## ไฟล์ที่เพิ่ม

```
src/app/[locale]/audit-logs/
├── page.tsx              # server: guard developer-only + query + filter + pagination
└── audit-log-table.tsx    # client: filter bar, expandable diff viewer
```

ทุกอย่างนี้ดึงข้อมูลจากตาราง `audit_logs` ที่ trigger สร้างให้อัตโนมัติอยู่แล้วตั้งแต่ Phase 1 — ไม่ต้องเขียน logic บันทึก log เพิ่มเลย แค่สร้างหน้าแสดงผล

---

ตอนนี้ทุกส่วนในสเปคเดิมเสร็จสมบูรณ์ครบ 100% แล้วครับ (รวมทั้ง Audit Logs ที่เพิ่งทำเสร็จ)

ทางเลือกต่อไป:
1. **Deploy ขึ้นจริง** — ผมช่วยตั้งค่า GitHub + Vercel ให้ทีละขั้นตอนได้
2. **Future Expansion** ตามสเปคเดิม — รูปภาพรถยนต์, แนบเอกสาร PDF, QR Code, แจ้งเตือนผ่าน LINE
3. **Import ข้อมูลเดิม** จากชีต Google Sheets ที่ใช้อยู่ตอนนี้ เข้าสู่ระบบใหม่
