# Phase 6 — สัญญาผ่อนชำระ (เสร็จแล้ว)

## สิ่งที่เพิ่มเข้ามา

**หน้ารายการสัญญา** `/[locale]/contracts`:
- ค้นหาเลขที่สัญญา, กรองตามสถานะ, pagination

**หน้าสร้างสัญญาใหม่** `/[locale]/contracts/new`:
- เลือกลูกค้า + เลือกรถยนต์ (แสดงเฉพาะคันที่สถานะ "พร้อมขาย" เท่านั้น)
- เลือกรถแล้วราคาขายจะถูกดึงมาใส่ให้อัตโนมัติ
- **ยอดจัดไฟแนนซ์คำนวณอัตโนมัติแบบเรียลไทม์** = ราคาขาย − เงินดาวน์ (ตรงกับ DB constraint ที่ตั้งไว้ใน Phase 1)
- เลขที่สัญญาออกให้อัตโนมัติ รูปแบบ `CT-[รหัสสาขา]-[ปีเดือน]-[เลขสุ่ม]`
- เมื่อบันทึกสำเร็จ: ระบบจะสร้าง**ตารางผ่อนชำระทั้งหมดอัตโนมัติ**ทันที (ใช้ trigger จาก Phase 1) และเปลี่ยนสถานะรถเป็น "อยู่ระหว่างผ่อน"
- ป้องกันการสร้างสัญญาซ้ำกับรถที่มีสัญญา active อยู่แล้ว (unique constraint) พร้อมข้อความแจ้งเตือน
- เข้าถึงได้จากหน้าโปรไฟล์ลูกค้าโดยตรง (ปุ่ม "สร้างสัญญาใหม่" จะ preselect ลูกค้าให้)

**หน้ารายละเอียดสัญญา** `/[locale]/contracts/[id]` — ใช้ฟังก์ชัน `get_contract_summary()` จาก Phase 1 คำนวณให้แบบเรียลไทม์:
- ยอดคงเหลือ (Remaining Balance)
- งวดที่ชำระแล้ว / งวดที่เหลือ
- งวดถัดไปครบกำหนดวันไหน
- ค้างชำระกี่วัน (ถ้ามี)
- ตารางผ่อนชำระทั้งหมดแบบละเอียด
- ปุ่มยกเลิกสัญญา (จะคืนสถานะรถเป็น "พร้อมขาย" อัตโนมัติ)

## ไฟล์ที่เพิ่ม

```
src/app/[locale]/contracts/
├── page.tsx                    # server: list + filter + pagination
├── contract-table.tsx            # client: filter bar, table
├── new/
│   ├── page.tsx                 # server: โหลดลูกค้า/รถที่พร้อมขาย
│   └── new-contract-form.tsx     # client: ฟอร์มสร้างสัญญา, คำนวณยอดจัดไฟแนนซ์สด
└── [id]/
    ├── page.tsx                 # server: ดึงสัญญา + เรียก get_contract_summary RPC
    └── detail-client.tsx          # client: การ์ดสรุป + ตารางผ่อนชำระ + ยกเลิกสัญญา
src/lib/supabase/contract-actions.ts   # server actions: สร้าง/ยกเลิกสัญญา
```

## ต่อไป: Phase 7 — การติดตามการชำระเงิน

Phase นี้สร้างตารางผ่อนชำระให้แล้ว แต่ยังไม่มีหน้าสำหรับ "บันทึกรับชำระเงิน" ของแต่ละงวด (เช่น พนักงานหน้าร้านรับเงินสด/โอน แล้วมาติ๊กว่าชำระแล้ว) — นั่นคือ Phase 7 ครับ พิมพ์ "continue" เพื่อไปต่อ
