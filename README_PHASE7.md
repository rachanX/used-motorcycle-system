# Phase 7 — บันทึกการชำระเงิน (เสร็จแล้ว)

## สิ่งที่เพิ่มเข้ามา

หน้า `/[locale]/payments` — มุมมองรวมของ "งวดที่ต้องเก็บเงิน" ข้ามทุกสัญญา ไม่ต้องเข้าไปทีละสัญญา:

- **ค้นหา** ด้วยเลขที่สัญญา
- **กรองตามสถานะ**: ชำระแล้ว / รอชำระ / ค้างชำระ
- **ปุ่มกรองด่วน**: ครบกำหนดวันนี้, ใกล้ครบกำหนด (7 วัน), เฉพาะค้างชำระ — ใช้ตอบคำถาม "วันนี้ต้องโทรตามใครบ้าง" ได้ทันที
- **บันทึกการชำระเงิน**: คลิกปุ่ม "บันทึกการชำระเงิน" ที่แต่ละแถว → กรอกยอดที่ชำระ (มีปุ่ม "ชำระเต็มจำนวน" ให้กดเร็ว), วันที่ชำระ, วิธีชำระ (เงินสด/โอน/QR/อื่นๆ), หมายเหตุ
- เมื่อบันทึกเสร็จ **สถานะงวดจะอัปเดตอัตโนมัติ** (ชำระแล้ว/รอชำระ/ค้างชำระ) และ **สถานะสัญญาก็อัปเดตตามไปด้วย** ผ่าน trigger ที่สร้างไว้ตั้งแต่ Phase 1 — ไม่มีโค้ดคำนวณซ้ำในชั้นแอป
- Staff เห็นเฉพาะงวดของสัญญาในสาขาตัวเอง (กรองที่ระดับ query และ RLS)

## ไฟล์ที่เพิ่ม

```
src/app/[locale]/payments/
├── page.tsx                    # server: join payments+contracts+customers, filter, pagination
├── payment-table.tsx            # client: filter bar, quick-filter chips, list
└── record-payment-modal.tsx     # client: form บันทึกการชำระเงิน
src/lib/supabase/payment-actions.ts   # server action: อัปเดตยอดชำระ/วันที่/วิธีชำระ
```

## ทำไมถึงไม่มีบั๊กเรื่องยอดไม่ตรง

แอปไม่เคยเขียนค่า `status` ของ payment หรือ `status` ของ contract เองเลย — เขียนแค่ `amount_paid`, `payment_date`, `payment_method`, `notes` แล้วปล่อยให้ trigger `refresh_payment_status` และ `sync_contract_status` (Phase 1) คำนวณสถานะให้ที่ฐานข้อมูลเสมอ จึงไม่มีทางที่ฐานข้อมูลกับหน้าจอจะแสดงสถานะไม่ตรงกัน
