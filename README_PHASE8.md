# Phase 8 — การแจ้งเตือน (เสร็จแล้ว)

## สิ่งที่เพิ่มเข้ามา

- **กระดิ่งแจ้งเตือน** บนแถบเมนู (ทั้งมือถือและเดสก์ท็อป) — แสดงตัวเลขจำนวนที่ยังไม่อ่าน, คลิกแล้วเห็น dropdown รายการล่าสุด 8 รายการ, คลิกแต่ละรายการเพื่อ mark-as-read ได้ทันที
- **หน้าการแจ้งเตือนแบบเต็ม** `/[locale]/notifications` — ดูได้ทั้งหมด 100 รายการล่าสุด, กรองตามประเภท (ครบกำหนดวันนี้/พรุ่งนี้/ภายใน 7 วัน, ค้างชำระ 1/3/7/30 วัน), ปุ่ม "อ่านทั้งหมดแล้ว"
- ทุกอย่าง branch-scoped เหมือนโมดูลอื่น — staff เห็นเฉพาะการแจ้งเตือนของสาขาตัวเอง
- เชื่อมกับฟังก์ชัน `generate_due_notifications()` ที่สร้างไว้ใน Phase 1 ซึ่งตั้งให้รันทุกวันผ่าน `pg_cron` (ฟรี ไม่ต้องใช้บริการเสริม) — ระบบจะมีแจ้งเตือนใหม่โผล่ขึ้นทุกเช้าโดยอัตโนมัติ ไม่ต้องมีคนกดปุ่มอะไร

## ไฟล์ที่เพิ่ม

```
src/components/notification-bell.tsx        # dropdown กระดิ่ง ใช้ในแถบเมนู
src/app/[locale]/notifications/
├── page.tsx              # server: ดึงรายการ + กรองตามประเภท
└── notification-list.tsx  # client: list เต็ม, ปุ่มอ่านทั้งหมด
src/lib/supabase/notification-actions.ts   # server actions: mark read / mark all read
```
แก้ไข `src/app/[locale]/layout.tsx` ให้ดึงจำนวนแจ้งเตือนที่ยังไม่อ่าน + รายการล่าสุดส่งให้ AppShell แสดงที่กระดิ่ง

## ตรวจสอบว่า cron ทำงานจริง

ใน Supabase SQL Editor รันคำสั่งนี้เพื่อดูว่า job ที่ตั้งไว้ใน Phase 1 (`daily-notifications`) ทำงานล่าสุดเมื่อไหร่:
```sql
select * from cron.job_run_details order by start_time desc limit 5;
```
